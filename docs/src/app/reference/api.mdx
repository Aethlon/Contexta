# API Reference

The HTTP API is the underlying wire protocol for the SDKs. Use the SDKs for development; use the API directly when an SDK is unavailable.

## Base URL

```
https://api.contexta.dev/v1
```

Regional endpoints:
- `https://eu.api.contexta.dev/v1` — EU pin
- `https://us.api.contexta.dev/v1` — US pin (available Q4 2026)

## Authentication

```
Authorization: Bearer mk_live_<32 url-safe bytes>
```

Dashboard server actions use `Authorization: contexta <session_jwt>`.

### API Key Scopes

| Scope | Access |
|-------|--------|
| `observations:write` | POST `/v1/observations` |
| `retrieval:read` | POST `/v1/retrieve`, GET `/v1/context` |
| `memories:read` | GET `/v1/memories/*` |
| `memories:write` | POST/PATCH/DELETE on `/v1/memories/*` |
| `policies:write` | Policy CRUD |
| `schemas:write` | Custom schema CRUD |
| `audit:read` | Audit log |
| `*` | All scopes |

Default scope: `observations:write,retrieval:read,memories:read`.

## Standard Headers

### Request
```
Authorization: Bearer mk_live_...
Content-Type: application/json
Idempotency-Key: <uuid>  (recommended for writes)
X-contexta-User-Id: <uuid> (mandatory for user-scoped endpoints)
```

### Response
```
X-contexta-Request-Id: 01J...
X-contexta-Region: eu-fsn1
X-contexta-Quota-Used: 0.42
X-contexta-RateLimit-Remaining: 47
X-contexta-RateLimit-Reset: 1716600000
```

## Observations

### POST /v1/observations

Submit a single observation.

```json
{
  "session_id": "33333333-3333-4333-8333-333333333333",
  "messages": [
    {"role": "user", "content": "I prefer Postgres over Mongo"},
    {"role": "assistant", "content": "Noted."}
  ],
  "metadata": {"agent": "coding-agent"},
  "policy": "coding-agent"
}
```

Response 202:
```json
{
  "job_id": "01J9ZXAB1Q2R3S4T5V6W7X8Y9Z",
  "status": "accepted",
  "estimated_processing_seconds": 3
}
```

### POST /v1/observations/batch

Submit up to 50 observations.

```json
{
  "observations": [{...}, {...}]
}
```

Response:
```json
{
  "jobs": [{"job_id": "...", "status": "accepted"}, ...]
}
```

Hybrid search.

```json
{
  "query_text": "what databases does the user prefer?",
  "session_id": "...",
  "memory_types": ["preference", "fact"],
  "tags": ["database"],
  "limit": 20,
  "graph_depth": 1,
  "include_archived": false,
  "include_cold": false,
  "rerank": false
}
```

Response:
```json
{
  "results": [
    {
      "memory_id": "...",
      "memory_type": "preference",
      "title": "Prefers Postgres",
      "content": "User prefers Postgres over Mongo.",

      "score_breakdown": {
```

## Memories

### GET /v1/memories/{id}

Full memory record.

### GET /v1/memories/{id}/explain

Explainability bundle: extraction source, scoring breakdown, supersession history.

### POST /v1/memories/{id}/pin

Pin a memory. Response:
```json
{"memory_id": "...", "is_pinned": true, "audit_event_id": "..."}
```

Same shape for `unpin`, `archive`, `restore`.

### DELETE /v1/memories/{id}

Hard delete. Returns 204.

### GET /v1/memories/timeline/{user_id}

Chronological memory events.

## Sessions

### POST /v1/sessions

Start a session (optional — auto-created on first observation).

### POST /v1/sessions/{id}/end

End session, triggers epilogue analysis.

### GET /v1/sessions/inspect/{user_id}

List recent sessions.

## Policies

### GET /v1/policies

List policies for tenant.

### POST /v1/policies

Register a custom policy.

```json
{
  "name": "support-agent",
  "store_rules": [
    {"memory_types": ["preference", "fact", "contact"], "patterns": ["customer.*"]}
  ],
  "ignore_rules": [{"patterns": ["^(thanks|hi|hello)$"]}],
  "priority_weights": {"preference": 0.9}
}
```

## Schemas

### POST /v1/schemas

Register a custom memory schema.

```json
{
  "name": "InvoiceLineItem",
  "field_definitions": [
    {"name": "vendor", "type": "string", "required": true},
    {"name": "amount", "type": "number", "required": true}
  ]
}
```

## Errors

```json
{
  "error": {
    "type": "validation_error",
    "code": "MISSING_FIELD",
    "message": "Field 'session_id' is required.",
    "fields": [{"field": "session_id", "message": "Field 'session_id' is required."}],
    "request_id": "01J9ZXAB1Q2R3S4T5V6W7X8Y9Z"
  }
}
```

| Error Type | HTTP | Cause |
|-----------|------|-------|
| `validation_error` | 422 | Bad payload |
| `authentication_error` | 401 | Missing/invalid bearer |
| `authorization_error` | 403 | Missing scope |
| `not_found` | 404 | Resource not found |
| `quota_exceeded` | 429 | Plan limit hit |
| `rate_limited` | 429 | RPS limit |
| `conflict` | 409 | Idempotency collision |
| `internal_error` | 500 | Unhandled error |
| `service_unavailable` | 503 | Infrastructure issue |

## Pagination

Cursor-based pagination on list endpoints:

```
GET /v1/memories?cursor=eyJpZCI6IjAxSjlaIn0&limit=50
```

Response includes `"next_cursor": "..."` or `null`.

## Versioning

- `/v1/` is locked. Breaking changes create `/v2/`.
- Non-breaking additions ship within `/v1/`.
- `/v1/` maintained 12 months after `/v2/` release.
